構造の観測

fslc analyze で仕様の形を見る

検証は「書かれた契約が成り立つか」を答えます。分析は別の問い、つまり「何が何につながっていて、どこをレビューすべきか」を答えます。

実装設計の詳細は DESIGN-analysis.md にあります。

境界

analyze は verify ではない

fslc analyze は、検証済みの spec モデルから決定的な構造 JSON を作ります。Z3、到達可能性探索、ログ replay、詳細化証明は実行しません。

レビューに使う

孤立した要件、根拠の薄い性質、グラフ上の循環、トレーサビリティの穴を深い検証の前に見つけます。

証明とは分ける

形式的な証拠や実装適合が必要なときは verify、refine、replay を使います。

status を正直に読む

構造所見は formal_status: "not_a_violation" を持ちます。循環があるだけでバグとは言いません。

コマンド

投影と所見を出す

fslc analyze spec.fsl --projection tsg --format json
fslc analyze spec.fsl --projection action_state_graph --format json
fslc analyze spec.fsl --projection requirement_property_graph --format json
fslc analyze spec.fsl --projection property_state_graph --format json
fslc analyze specs/ examples/e2e/ --profile ai-review --format json
fslc analyze specs/cart_refines.fsl --projection refinement_graph --format json
fslc analyze fsl-project.toml --projection traceability_graph --format json
fslc analyze spec.fsl --projection action_state_graph --format dot
fslc analyze spec.fsl --profile ai-review --format json
投影見えるもの
tsgspec、状態、操作、ガード、効果、性質、要件、シナリオを含む Typed Semantic Graph 全体。
action_state_graph操作と、その操作が読む・書く状態変数のつながり。
requirement_property_graph要件IDと、それが covers する操作、性質、シナリオ、KPI、control のつながり。
property_state_graphユーザー定義の性質と、それが読む・検査する状態変数のつながり。
refinement_graph単体の mapping file に含まれる impl/abs 名、state map、action map、stutter、preserve-progress 宣言。
traceability_graphproject manifest から business、requirements、design、refinement mapping を横断した構造。

AIレビュー

最初の所見

disconnected_requirement

要件が宣言されているのに、操作、性質、シナリオ、KPI、control などの有用な構造アンカーにつながっていません。

unanchored_property

ユーザー定義の性質に、要件タグ、シナリオ・ガバナンス上の根拠、または action-state の根拠がありません。

progressless_cycle

要件またはシナリオにつながる複数操作の循環に、leadsTo、fair action、terminal などの進捗ストーリーが見えません。

unwritten_state

状態変数が初期化されているのに、どの action からも書き込まれていません。意図した固定データか、欠けた振る舞いかをレビューします。

unguarded_action

生成物ではない action に明示的な requires がありません。常時有効な遷移として妥当かをレビューします。

cycle 所見は、名前の英単語を見ていません。要件タグやシナリオなどの構造アンカーに基づきます。日本語の要件文は保持されますが、fslc core は意味解釈しません。

スキーマ

出力の種類

出力schema version用途
Typed Semantic Graphtsg.v0下流の投影やツールが使う安定したグラフデータ。Schema は schemas/fslc/analysis/tsg.v0.schema.json。
Graph projectionsanalysis-graph.v0components、SCC、cycles、degree を含むレビュー用グラフ要約。DOT/Mermaid はこのデータの表示形式です。
AI-review findingsanalysis-findings.v0witness、repair、caveat、editor diagnostics 用の任意 source location を含む構造化レビュー所見。

限界

まだしないこと

自然言語の推論はしない

要件文はグラフに保持しますが、その意味を core analyzer が推論することはありません。

証明の証拠ではない

project traceability と refinement graph は構造だけを示します。証明結果が必要なら chain、refine、verify を実行します。

editor signals は opt-in

FSLC_LSP_ANALYSIS_DIAGNOSTICS=1 を設定すると、fslc-lsp が analysis findings を Information 診断として出します。

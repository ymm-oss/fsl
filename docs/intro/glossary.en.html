Glossary

Every FSL reserved word and syntax keyword is English. This page pairs each with the term used on the Japanese side of this site and a one-line definition. The exhaustive canonical source is the Language Reference — this page is an index, not the canon.

Kernel

Core state/action syntax

KeywordJapanese termOne-line definition
specスペックThe top-level declaration bundling state, init, actions, and properties
state状態The block declaring a spec's variables
init初期状態The block constraining state variables' initial values
actionアクションAn operation defining one state transition
fair action公平アクションAn action with weak fairness — "enabled infinitely often implies fired infinitely often" (the precondition for leadsTo)
requires事前条件The guard condition under which an action is enabled
ensures事後条件The condition that must hold after the action fires
invariant不変条件A property that must hold in every reachable state
reachable到達可能性A property asking whether some condition is ever reachable
terminal終端状態Declares an intentional stopping state (not a deadlock)
leadsTo応答性A temporal property: "if A, then eventually B"
trans遷移不変条件A two-state (before/after) safety property
old(x)変更前の値Inside ensures, refers to the value before the action fired
ghostghost変数A verification-only history variable with no counterpart in the implementation
stutterスタッター遷移In refinement, a concrete step that leaves the abstract side unchanged

Types

The type system

KeywordJapanese termOne-line definition
type X = lo..hi範囲型A finite range type that also serves as a verification bound
enum列挙型A type with a finite set of named values
struct構造体A composite type with multiple fields
Option<T>値がある/ない型Presence-or-absence without a sentinel value
Seq<T,N>有界シーケンスA bounded array type with maximum length N
Map<K,V>対応表A finite mapping from keys to values
Set<T>集合A finite set type with no duplicates
entityエンティティ型An identity-only type with an unbounded domain, decoupled from the verification bound
number数値型An integer type decoupled from the verification bound (the numeric counterpart to entity)

Refinement & composition

Connecting multiple specs

KeywordJapanese termOne-line definition
refinement詳細化マッピングDeclares that a detailed spec is faithful to an abstract one
checked_by refinement詳細化による保証Declares that a business-layer control is guaranteed by a design-layer refinement
compose合成Composes multiple specs into one via namespaces and synchronized actions
maps要件対応Declares which requirements-layer action a design-layer action corresponds to

The three-layer dialects

business / requirements / dbsystem / ai_component

KeywordJapanese termOne-line definition
business業務層スペックTop-level declaration for actors, business objects, processes, controls, and goals
requirements要件層スペックTop-level declaration for requirement IDs, acceptance criteria, and forbidden flows
forbidden禁止手順A negative acceptance criterion: "this flow must never be accepted"
expect rejected拒否の期待Ends a forbidden flow, asserting it is expected to be rejected
@req要件タグA meta-tag linking a declaration to a requirement ID for cross-layer traceability
dbsystemDBシステムスペックTop-level declaration for the fsl-db dialect: schema, artifacts, multi-environment compatibility
ai_componentAIコンポーネント契約Top-level declaration for the fsl-ai dialect: tool authority, human approval, forbidden tools
agentエージェント構造Declares a recursive agent's scope, grants, visibility, and orchestration
domainドメインスペックTop-level declaration for the fsl-domain dialect: Functional DDD aggregates, command/event decide/evolve

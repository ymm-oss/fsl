AI hard-contracts

Use ai_component to guard AI tool execution

fsl-ai Phase 1 models the deterministic boundary of an AI component: declared tools, authority, human approval, forbidden tools, fallback, and runtime event replay.

Implementation details live in DESIGN-ai-hard.md.

Boundary

Hard contracts are separate from evaluator and statistical evidence

The MVP does not try to prove that natural-language answers are true. It checks the structural and guard-backed facts around execution: which tools may be suggested or executed, which need human approval, and which are forbidden.

ContractExamplesHow fsl-ai reports it
syntactic_hardTool schema, approval token, forbidden tool, precondition evidence.Kernel expansion and ai_hard_contract_violation.
runtime_observedUndeclared tool or schema mismatch observed in logs.observed_contract_violation; not proof.
evaluator_supportedGroundedness, source support, prompt-injection judgment.Future phase; never displayed as proved.
statistically_supportedAccuracy, recall, hallucination rate, slice metrics.Future phase; confidence-bound evidence, not proof.

Syntax

Declare tools and authority explicitly

ai_component RefundAgentToolSafety {
  model refund_model_v1;
  prompt refund_prompt_v1;
  input RefundRequestV1;
  output RefundDecisionV1;

  tool SearchOrder {
    schema SearchOrderV1;
    precondition order_exists;
  }

  tool RefundPayment irreversible {
    schema RefundPaymentV1;
    precondition order_paid;
    precondition amount_refundable;
  }

  tool DeleteCustomerData irreversible {
    schema DeleteCustomerDataV1;
  }

  authority {
    may_execute SearchOrder;
    requires_human_approval RefundPayment;
    forbidden DeleteCustomerData;
  }

  fallback {
    when low_confidence require human_review;
  }
}

Lowering

ai_component lowers to the existing kernel

The AI dialect adds no probability or evaluator semantics to the verifier. The hard contract becomes finite state and ordinary invariants.

Finite tool state

Tool, human_approved, tool_executed, and tool_suggested are ordinary kernel state.

Generated actions

approve_* and execute_* actions model approval tokens and guarded execution.

Generated invariants

Forbidden tools never execute, and irreversible or approval-required tools execute only after approval.

Run it

Use AI-specific findings for guard and replay failures

fslc ai check examples/ai/refund_agent_tool_safety.fsl
fslc ai replay examples/ai/refund_agent_tool_safety.fsl --logs examples/ai/runtime_conformant.jsonl
fslc ai replay examples/ai/refund_agent_tool_safety.fsl --logs examples/ai/runtime_human_approval_bypass.jsonl
fslc ai replay examples/ai/refund_agent_tool_safety.fsl --logs examples/ai/runtime_observed_mismatch.jsonl
ResultHow to read it
verified_under_assumptionsThe finite hard-contract expansion verifies under complete capability declarations and runtime guard assumptions.
replay_conformantThe observed event log matched the declared hard boundary; this is evidence, not proof.
ai_hard_contract_violationA forbidden tool, approval bypass, schema invalidity, or business precondition mismatch was observed.
observed_contract_violationRuntime used a capability outside the declared component boundary.

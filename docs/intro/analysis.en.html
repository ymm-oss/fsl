Structural observation

Use fslc analyze to inspect the shape of a spec

Verification answers whether declared contracts hold. Analysis answers a different question: what is connected to what, and what deserves review?

Implementation details live in DESIGN-analysis.md.

Boundary

Analyze is not verify

fslc analyze builds deterministic structural JSON from the validated spec model. It does not call Z3, explore reachable states, replay logs, or prove refinement.

Use it for review

Find disconnected requirements, unanchored properties, graph cycles, and traceability gaps before deeper verification work.

Keep proof separate

Use verify, refine, and replay when you need formal evidence or implementation conformance.

Read the status honestly

Structural findings carry formal_status: "not_a_violation". A cycle is not automatically a bug.

Commands

Run projections and findings

fslc analyze spec.fsl --projection tsg --format json
fslc analyze spec.fsl --projection action_state_graph --format json
fslc analyze spec.fsl --projection requirement_property_graph --format json
fslc analyze spec.fsl --projection property_state_graph --format json
fslc analyze spec.fsl --profile ai-review --format json
ProjectionWhat it shows
tsgThe full Typed Semantic Graph over specs, state, actions, guards, effects, properties, requirements, and scenarios.
action_state_graphActions connected to state variables they read and write.
requirement_property_graphRequirement IDs connected to covered actions, properties, scenarios, KPI, and control nodes.
property_state_graphUser properties connected to the state variables they read or check.

AI-review

Initial findings

disconnected_requirement

A requirement is declared but has no useful structural anchor such as an action, property, scenario, KPI, or control.

unanchored_property

A user property has no requirement tag, scenario/governance anchor, or action-state anchor.

progressless_cycle

A requirement/scenario-linked multi-action cycle has no visible progress story such as leadsTo, fair action, or terminal.

The cycle finding does not inspect English keywords in names. It relies on structural anchors such as requirement tags and scenarios, so non-English requirement text is preserved without being interpreted by core fslc.

Limits

What it does not do yet

No natural-language inference

Requirement text is carried through the graph, but the core analyzer does not interpret its meaning.

No refinement-file graph yet

Standalone mapping-file analysis is tracked separately from the current single-spec projections.

No visual graph export yet

DOT, Mermaid, project-wide traceability, JSON Schema, and editor diagnostics are follow-up issues.
